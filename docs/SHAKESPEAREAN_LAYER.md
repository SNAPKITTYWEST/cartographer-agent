# Shakespearean Layer

## Purpose

This layer is not the enforcement engine. It is the rhetorical and ceremonial legal surface.

The operative law-team stack remains:

- corpus
- kernel team
- reasoning fabric
- trust deed governance

The Shakespearean layer sits above that as presentation and composition.

## Why It Exists

- British legal rhetoric has a formal cadence that fits the law-team aesthetic
- it gives the system a distinctive drafting voice
- it creates a ceremonial code surface without altering the underlying authority model

## What It Should Do

- render legal actions in stylized declarative form
- present summons, deeds, and governance moves as dramatic clauses
- remain readable to humans even when it is stylized

## What It Should Not Do

- replace Prolog authority facts
- replace deed governance
- become the only representation of operative state

## Example

```text
When CARTO doth survey the matter,
let the kernel be chosen by the weight of law.

If the dispute soundeth in standing and breach,
send it unto Prolog.

If the work be packet, ledger, or notice,
send it unto COBOL.

If the office requireth motion and route,
send it unto REXX.

If authority be doubted,
raise the Trust Deed before all action.
```

## Placement In The Stack

1. operator enters matter
2. Shakespearean layer frames the matter
3. kernel / fabric / deed layers perform the real work
4. WORM seal records the authoritative path
